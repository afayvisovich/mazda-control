# 🔍 Диагностика сети без root прав

## Проблема
Сервер **172.16.2.30:50001** недоступен. Системный AG35TspClient тоже не может подключиться.

---

## ✅ Что добавлено

### 1. **Кнопка диагностики в приложении**

В приложении появилась новая кнопка:
```
🔍 Диагностика сети (172.16.2.30:50001)
```

**Что делает:**
- Проверяет доступность сервера через ping
- Проверяет порт 50001 через TCP socket
- Показывает все сетевые интерфейсы
- Показывает активную сеть
- Работает **без root прав**

**Результат:**
```

═══════════════════════════════════════
📍 Сервер: 172.16.2.30:50001
⏱️ Время проверки: ...

🌐 Активная сеть: Ethernet

📋 Сетевые интерфейсы:
  • bridge2 (bridge2): 172.16.2.2 ↑
  • eth0.3 (eth0.3): 172.16.3.2 ↑
  • wlan0 (wlan0): 10.249.68.36 ↑
  ...

📍 Результаты:
  ❌ Ping: false
  ❌ Порт 50001: false

🎯 ИТОГ: СЕРВЕР НЕДОСТУПЕН
```

### 2. **Скрипты для ПК**

#### `diagnostic_no_root.sh` - полная диагностика
```bash
./diagnostic_no_root.sh > network_diagnostic.txt
```

**Что проверяет:**
- IP адреса всех интерфейсов
- Таблица маршрутизации
- ARP таблица (соседи)
- Ping шлюза 172.16.2.1
- Ping 172.16.2.30
- Активные TCP подключения
- Слушающие порты

#### `scan_network.sh` - сканирование сети
```bash
./scan_network.sh
```

**Что делает:**
- Сканирует диапазон 172.16.2.1-50
- Ищет активные хосты
- Проверяет порт 50001 на найденных IP

## 🎯 Порядок тестирования

### Шаг 1: Запустить диагностику в приложении

1. Установить приложение:
   ```bash
   adb install -r app/build/outputs/apk/release/app-release.apk
   ```

2. Открыть приложение

3. Нажать кнопку **"🔍 Диагностика сети (172.16.2.30:50001)"**

4. Посмотреть результат в логе

---

### Шаг 2: Запустить скрипты на ПК

```bash
# Полная диагностика
./diagnostic_no_root.sh

# Сканирование сети
./scan_network.sh
```

---

### Шаг 3: Проанализировать результаты

**Если ping не проходит:**
- Сервер выключен
- Неверный IP адрес
- Firewall блокирует ICMP

**Если ping проходит, но порт закрыт:**
- Сервер работает на другом порту
- Firewall блокирует порт 50001

**Если оба теста провалены:**
- Искать сервер в другой подсети
- Проверить физическое подключение

## 🔧 Команды для ручной проверки (без root)

```bash
# Ping сервера
adb shell ping -c 5 172.16.2.30

# Посмотреть IP адреса
adb shell ip addr show

# Таблица маршрутизации
adb shell ip route

# ARP таблица
adb shell ip neigh

# Netstat (без PID)
adb shell netstat -an | head -50
```

## 📊 Формат отчёта

Отчёт диагностики включает:

1. **Информация о сервере**
   - Хост и порт
   - Время проверки

2. **Сетевые интерфейсы**
   - Имя интерфейса
   - IP адрес
   - Статус (up/down)

3. **Результаты проверки**
   - Ping (ICMP)
   - Порт (TCP socket)

4. **Итоговый вердикт**
   - ДОСТУПЕН / НЕДОСТУПЕН

---

## ⚠️ Ограничения

- **Нет root** → некоторые команды недоступны
- **Нет bash** → проверка портов через Kotlin socket
- **Android 11+** → ограничения доступа к сети

**Решение:**
- Используем `InetAddress.isReachable()` для ping
- Используем `Socket.connect()` для проверки портов
- Работает на всех Android 11+ без root
