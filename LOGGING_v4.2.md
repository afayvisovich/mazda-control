## 🔍 Как использовать логи

### 1. **Через файловый менеджер на устройстве**
```

  ├── tbox_log-20260315_143022.txt
  └── mazda_log-20260315_143022.txt
```

### 2. **Через приложение**
- Нажмите кнопку **"💾 Сохранить лог"**
- Выберите приложение для отправки

### 3. **Через ADB (если нужно)**
```bash
adb pull /sdcard/Download/MazdaControl/
```

## 📊 Структура лога

### Заголовок пакета:
- **Байт 0:** Magic byte `0x5A`
- **Байты 1-3:** Версия/тип
- **Байты 4-7:** Длина тела (Little Endian)
- **Байты 8-13:** Reserved

### Тело пакета:
- **Байты 288-295:** Function Bytes (команда)
- **Байт 355:** Статус/подтверждение (`0x2B`)
- **Байт 363:** Синхронизация (`0x5A`)

### CRC16:
- Последние 2 байта пакета
- Проверка целостности

## 🔧 Отладка проблем

### Если лог пустой:
1. Проверьте разрешения приложения
2. Убедитесь, что папка создана: `/sdcard/Download/MazdaControl/`
3. Проверьте логи через `adb logcat | grep TBoxSpoilerController`

### Если CRC ошибка:
- Смотрите в логе: `CRC=FAIL`
- Проверьте целостность пакета
- Сравните с оригинальными логами автомобиля

### Если пакет не приходит:
- Проверьте подключение: `127.0.0.1:32960`
- Запустите диагностику сети
- Смотрите логи подключения
