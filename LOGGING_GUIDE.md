## Элементы лога

### Общие элементы

| Элемент | Описание |
|---------|----------|
| `🏷️ Action` | Название команды (OPEN, CLOSE, FOLLOW_SPEED, SPORT_MODE) |
| `📦 ... bytes` | Размер данных |
| `🔢 Hex dump` | Дамп в hex формате с ASCII представлением |
| `🚀 Отправка...` | Начало отправки |
| `✅ ... за Xms` | Время отправки |

### JSON mode специфичные

| Элемент | Описание |
|---------|----------|
| `📤 ОТПРАВКА JSON КОМАНДЫ` | Заголовок лога JSON режима |
| `📄 JSON Payload` | JSON строка |
| `📦 JSON bytes` | Размер JSON в байтах |
| `🎯 КЛЮЧЕВЫЕ ПОЛЯ JSON` | Распарсенные поля Payload |
| `• value: X (NAME)` | Команда с названием |
| `• time: X (YYYY-MM-DD HH:MM:SS.SSS)` | Timestamp в читаемом виде |

### TBox mode специфичные

| Элемент | Описание |
|---------|----------|
| `📤 ОТПРАВКА TBox ПАКЕТА` | Заголовок лога TBox режима |
| `📋 ЗАГОЛОВОК (30 байт)` | Заголовок пакета |
| `📦 ТЕЛО ПАКЕТА (412 байт)` | Тело пакета |
| `✅ CRC16: XX XX` | Контрольная сумма |
| `🎯 КЛЮЧЕВЫЕ БАЙТЫ` | Ключевые позиции для спойлера |
| `Function Bytes [288-295]` | Байты функции |
| `Byte 355` | Байт состояния 1 |
| `Byte 363` | Байт состояния 2 |

## Значения value

| Value | Name | Description |
|-------|------|-------------|
| 1 | OPEN | Открыть спойлер |
| 2 | CLOSE | Закрыть спойлер |
| 3 | FOLLOW_SPEED | Режим "По скорости" |
| 4 | SPORT_MODE | Спортивный режим |

## Формат времени

```
time: 1234567890 (2024-01-15 10:30:45.123)
       ↑            ↑
       │            └─ Человекочитаемый формат
       └─ Timestamp в миллисекундах
```

## Поиск в логах

```bash
# Все логи контроллера
adb logcat | grep TBoxSpoilerController

# Только JSON команды
adb logcat | grep "ОТПРАВКА JSON"

# Только TBox пакеты
adb logcat | grep "ОТПРАВКА TBox"

# Только ошибки
adb logcat | grep "❌"

# Время отправки
adb logcat | grep "отправлен за"
```

## Типичные проблемы

### ❌ Connection refused

```
❌ Ошибка отправки: Connection refused
```
**Решение:** Fake32960Server не запущен

### ❌ Timeout

```
✅ JSON отправлен за 5000ms
```
**Решение:** Сервер не отвечает, проверить сокет

### ⚠️ Failed to parse JSON

```
⚠️ Failed to parse JSON fields: ...
```
**Решение:** Неверный формат JSON, проверить структуру

## Советы по отладке

1. **Включите полный лог**
   ```kotlin
   Log.d(TAG, "=== SENDING JSON COMMAND ===")
   ```

2. **Смотрите hex dump**
   ```
   🔢 Hex dump (79 bytes):
   ```

3. **Проверяйте ключевые поля**
   ```
   🎯 КЛЮЧЕВЫЕ ПОЛЯ JSON:
     • value: 1 (OPEN)  ← Должно быть 1-4
     • valid: true      ← Должно быть true
   ```

4. **Контролируйте время отправки**
   ```
   ✅ JSON отправлен за 5ms  ← Норма: <50ms
   ```

5. **Сравнивайте режимы**
   ```
   JSON: ~80 bytes, 5ms
   TBox: 444 bytes, 12ms
   ```
