### 2. Обработка ошибок GB32960 Protocol

**Документ:** `04-optional/ERROR_HANDLING_ANALYSIS.md`

**Типы некорректных данных и реакция сервера:**

| Тип ошибки | Реакция сервера | Влияние на спойлер |
|------------|-----------------|-------------------|
| **Неверные Function Bytes** | Игнорирование | ❌ Тишина |
| **Неверный VIN** | Отклонение | ❌ Тишина |
| **Неверная checksum** | Отклонение | ❌ Error в логе |
| **Неверный timestamp** | Отклонение | ❌ Старые пакеты |
| **Команда другому компоненту** | Выполнение | 🪟 Другое действие |
| **Недопустимое значение** | Игнорирование | ❌ Тишина |
| **Нарушение безопасности** | Блокировка | ❌ Safety block |

**Уровни защиты:**
1. **Парсинг** → длина, VIN, checksum, timestamp
2. **Валидация** → Component ID, Action Code, диапазон значений
3. **Безопасность** → скорость, передача, состояние компонента

**Вывод:** Сервер устойчив к ошибкам, но лучше отправлять корректные данные!

### 3. VIN автомобиля

**Статус:** ✅ **ИСПОЛЬЗУЕТСЯ В PacketGenerator.kt**

**VIN:** `LVRHDAEJ8RN013579` (17 символов)

**Кодирование в пакете:**
```kotlin
// Байты 4-20 в заголовке (17 байт VIN)
private val VIN = byteArrayOf(
    0x4C, 0x56, 0x52, 0x48, 0x44, 0x41, 0x45, 0x4A,
    0x38, 0x52, 0x4E, 0x30, 0x31, 0x33, 0x35, 0x37, 0x39
)
```

**Важность:**
- ✅ **Валидируется сервером** — если отправить с чужим VIN, сервер отклонит пакет
- ✅ **Специфичен для автомобиля** — приложение работает только с твоим авто
- ⚠️ **Безопасность** — VIN не является секретом, но лучше не коммитить в репозиторий

**Рекомендация:** Текущий вариант корректный, но можно вынести в `keystore.properties` или настройки.

## 🎯 Единственный способ управления: GB32960 Protocol

**Механизм:** Отправка 512-байтных пакетов через localhost:32960

**Команды:**

| Действие | Function Bytes | Byte 355 | Byte 363 | Byte 365 |
|----------|----------------|----------|----------|----------|
| **OPEN** | `BF 11 0D 06 27 21 20 01` | `0x2B` | `0x5A` | `0x00` |
| **CLOSE** | `BF 11 0D 05 27 23 20 01` | `0x2B` | `0x5A` | `0x00` |

**Архитектура:**
```
MazdaControl App
       ↓
Socket localhost:32960
       ↓
com.mega.rmu (PID 8895) — RMU Service
       ↓
CAN-шина → T-Box (172.16.2.30:49960)
```

**Преимущества:**
- ✅ Полный контроль над таймингами
- ✅ Работает напрямую с CAN-шиной
- ✅ Независимо от других сервисов
- ✅ Документировано в `reverse-engineering.md`

**Недостатки:**
- ❌ Низкоуровневый протокол
- ❌ Нужно самому отслеживать состояния
- ❌ Нет встроенных проверок безопасности
- ❌ Риск конфликтов с штатной системой

**Файлы:**
- `app/src/main/java/com/mazda/control/PacketGenerator.kt`
- `app/src/main/java/com/mazda/control/SpoilerController.kt`

### Способ 2: CarProperty API (высокоуровневый)

**Механизм:** Использование штатного MegaCarProperty API

**Property ID:**

| ID | Hex | Назначение | Значения |
|----|-----|------------|----------|
| `BODYINFO_HU_SPOILERSWITCH` | `0x660000c3` | Команда | `1`=открыть, `2`=закрыть |
| `BODYINFO_PTS_SPOILERPOSITION` | `0x660000c4` | Позиция | `101`=открыт |
| `BODYINFO_PTS_SPOILERMOVEMENT` | `0x660000c5` | Движение | `0`=нет, `1`=движется, `2`=готово |
| `LIGHT_HU_SPOILERWELCOMESWITCH` | `0x660000d1` | Приветствие | Световое шоу |

**Архитектура:**
```
MazdaControl App
       ↓
MegaCarProperty.setIntProp()
       ↓
HuronCarService
       ↓
CAN-шина
```

**Преимущества:**
- ✅ Простой API (одна строка кода)
- ✅ Автоматическая обработка состояний
- ✅ Встроенные проверки безопасности
- ✅ Event callbacks для обновления UI
- ✅ Документировано в `CARBODY_CONTROLLER_PACKAGE.md`

**Недостатки:**
- ❌ Зависимость от HuronCarService
- ❌ Может быть недоступно в фоновом режиме
- ❌ Меньше контроля над таймингами

**Пример кода:**
```kotlin
// Открыть спойлер
MegaCarProperty.getInstance().setIntProp(0x660000c3, 1)

// Закрыть спойлер
MegaCarProperty.getInstance().setIntProp(0x660000c3, 2)

// Проверить позицию
val position = MegaCarProperty.getInstance().getIntProp(0x660000c4)

// Подписка на события
val callback = object : CarPropertyEventCallback {
    override fun onEvent(prop: CarProperty) {
        when (prop.propertyId) {
            0x660000c4 -> updatePosition(prop.getIntValue())
            0x660000c5 -> updateMovement(prop.getIntValue())
        }
    }
}
```

## 🔍 Новые знания (14 марта)

### 1. Архитектура систем автомобиля

**HU (Head Unit)** — головное устройство:
- Отправляет команду на спойлер (`0x660000c3`)
- Интерфейс для водителя

**PTS (Power Tailgate System)** — система управления багажником:
- Возвращает позицию (`0x660000c4`)
- Возвращает статус движения (`0x660000c5`)

**Light System** — система освещения:
- Приветственный режим (`0x660000d1`)
- Световое шоу со спойлером

### 2. Конфигурация автомобиля

**ECU Property:**
- `ID_CONFIG_ELECTRIC_REAR_SPOILER` = `0x3310025a`
- System Property: `ro.ecu.config.ELECTRIC_REAR_SPOILER`
- Значение: `1` = спойлер есть, `0` = нет

**Проверка перед управлением:**
```java
int hasSpoiler = MegaSystemProperties.getInt("ro.ecu.config.ELECTRIC_TAIL", -1);
if (hasSpoiler != 1) {
    // Спойлер отсутствует
    return;
}
```

### 3. Центральная сервисная архитектура

**MegaControllerService** — главный сервис:
- Инициализирует **28 контроллеров**
- Предоставляет **3 AIDL интерфейса**
- Регистрируется на **TLV события** (`0x30000001`)
- HandlerThread для фоновой работы

**TLV Protocol** — универсальный механизм:
- Используется для диагностики (7 систем)
- Используется для уведомлений о зарядке
- Общий транспорт: `0x30000001` → `MegaCarProperty.setRawProp()`

### 4. Система диагностики

**VehicleHealthController** — 7 проверок:
- TPMS, Airbag, EPS, ESC, Motor, ABS
- Remote Diagnosis через TLV
- Отправка в T-Box через 50ms debounce

**Не относится к спойлеру**, но показывает общую архитектуру TLV.

## 🚫 Почему CarProperty API не работает

### Архитектура доступа к свойствам автомобиля:

```
┌─────────────────────────────────────────┐
│  Системные приложения                   │
│  (android.uid.system)                   │
│  ┌─────────────────────────────────┐    │
│  │ HuronCarController              │    │
│  │ (com.mega.controller)           │    │
│  │                                 │    │
│  │ CarPropertyManager              │    │
│  │ ↓                               │    │
│  │ ICarProperty (AIDL)             │    │
│  │ ↓                               │    │
│  │ MegaCarProperty                 │    │
│  └─────────────────────────────────┘    │
│           ↓                             │
│        CAN-шина                         │
└─────────────────────────────────────────┘
           ↑
    ════════════════
    СИСРЕМНЫЙ БАРЬЕР
    ════════════════
           ↓
┌─────────────────────────────────────────┐
│  Сторонние приложения                   │
│  (обычная подпись)                      │
│  ┌─────────────────────────────────┐    │
│  │ MazdaControl                    │    │
│  │ (com.mazda.control)             │    │
│  │                                 │    │
│  │ ❌ Нет доступа к                │    │
│  │    CarPropertyManager           │    │
│  │ ❌ Нет доступа к                │    │
│  │    ICarProperty                 │    │
│  │ ❌ Нет доступа к                │    │
│  │    MegaCarProperty              │    │
│  │                                 │    │
│  │ ✅ Socket localhost:32960       │    │
│  │ ✅ GB32960 Protocol             │    │
│  └─────────────────────────────────┘    │
│           ↓                             │
│        CAN-шина (через RMU)             │
└─────────────────────────────────────────┘
```

### Почему это важно:

1. **Безопасность Android:**
   - Системные API защищены permission'ами
   - `android.car.hardware` требует системной подписи
   - Обычные приложения не могут вызвать `CarPropertyManager`

2. **MegaCarProperty — внутренний класс:**
   - Находится в пакете `com.mega.controller`
   - Использует системный AIDL сервис
   - Недоступен извне

3. **Единственный легальный способ:**
   - GB32960 Protocol через сокет
   - Работает на любом приложении
   - Не требует привилегий

**Проблема:** Сервер, который слушает порт 32960, **НЕ найден** в декомпилированных APK:
- ❌ HuronCarService.apk
- ❌ HuronCarController.apk

**Известно:**
- CallServerInterceptor (OkHttp) отправляет пакеты на `localhost:32960`
- Это **TCP socket** (подтверждено логами)
- Сервер отвечает статус-пакетами (mark=3)

**Требуется извлечь дополнительные APK:**
1. **com.mega.rmu** (Remote Management Unit) — наиболее вероятный кандидат
2. **com.mega.gateway** (Gateway Service) — центральный сервер
3. **com.mega.car.tbox** (TBox Service) — обработчик GB32960

**См.:** `04-optional/APK_EXTRACTION_REQUEST.md`

### 4. Если GB32960 не работает:
