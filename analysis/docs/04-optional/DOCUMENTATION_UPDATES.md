# Обновления документации - Март 2026

## Резюме изменений в ANALYSIS.md

### ✅ Добавленные данные

#### 1. Counter (Byte 355)
- **Что выявлено:** Counter уменьшается на 1 каждую секунду при polling
- **Пример:** `0xB7` (183) → `0xB6` (182) → `0xB5` (181)
- **Источник:** `logs/sp_op2_cl_op_cl_op3.txt`

#### 2. Command Mark Type (Byte 363)
- **`0x5A`** = Исходящая команда (от приложения к серверу)
- **`0x0A`** = Входящий ответ (от сервера приложению)
- **Источник:** `logs/sp_op2_cl_op_cl_op3.txt`, `logs/applog.txt`

#### 3. Time Byte 29 (Action Code)
- **`0x02`** = OPEN/ON (открытие/включение)
- **`0x0C`** = CLOSE/OFF (закрытие/выключение)
- **`0x16`** = 50% позиция (для окон)
- **Источник:** Анализ всех команд в логах

#### 4. Процессы автомобиля
- **`com.mega.car`** (PID 6624) - Основное приложение автомобиля
- **`com.mega.controller`** (PID 7414) - Контроллер компонентов
- **`com.mega.carmodel`** (PID 6702) - 3D модель автомобиля
- **Источник:** `logs/psa.txt`

#### 5. Порт подключения
- **`localhost:32960`** - Локальный сервер команд
- **Требуется:** `adb forward tcp:32960 tcp:32960` для отладки
- **Источник:** `logs/applog.txt`, `logs/sp_op2_cl_op_cl_op3.txt`

## 🔍 Выявленные проблемы

### 1. NetworkOnMainThreadException
**Проблема:** Приложение пытается подключиться к сокету в UI потоке
**Решение:** Использовать асинхронное подключение (Coroutines/Thread)

### 2. Отсутствие логов исходящих команд
**Проблема:** В логах видны только входящие ответы (`0x0A`), но не команды (`0x5A`)
**Решение:** Использовать расширенное логирование или tcpdump

## 🎯 Следующие шаги

### Для сбора логов исходящих команд:

```bash
# 1. Установить verbose логирование
adb shell setprop log.tag.CallServerInterceptor VERBOSE
adb shell setprop log.tag.RealTaskPipeline VERBOSE

# 2. Запустить логирование
adb logcat -v time -s CallServerInterceptor:I > outgoing.txt

# 3. Выполнить команды спойлером в приложении

# 4. Анализировать строки с "request <3" и "<363>: 5a"
```

### Для отладки подключения:

```bash
# 1. Настроить port forwarding
adb forward tcp:32960 tcp:32960

# 2. Проверить доступность порта
adb shell netstat -tuln | grep 32960

# 3. Перехватить трафик (требуется root)
adb shell "tcpdump -i lo -s 0 -w /data/local/tmp/spoiler.pcap port 32960"
```
