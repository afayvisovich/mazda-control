# ⚠️ Анализ ошибок GB32960 Protocol

**Дата:** 2026-03-14  
**Вопрос:** Что будет при отправке некорректного кода?  
**Статус:** ✅ **АНАЛИЗ ЗАВЕРШЁН**

---

## 🎯 Типы некорректных данных

### 1. Некорректные Function Bytes

**Пример:** Отправка несуществующей команды
```kotlin
// Вместо правильной команды спойлеру:
val correctOpen = byteArrayOf(0xBF, 0x11, 0x0D, 0x06, 0x27, 0x21, 0x20, 0x01)

// Отправляем мусор:
val invalid = byteArrayOf(0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF)
```

**Что произойдёт:**
- ✅ **Сервер проигнорирует пакет** (не распознанная команда)
- ✅ **Никакой реакции** от спойлера
- ✅ **В логе:** `ResponseParserInterceptor: unknown command`
- ❌ **Без ошибок** — просто тишина

---

### 2. Некорректный VIN (bytes 0-20)

**Пример:** Неправильный VIN
```kotlin
// Правильный VIN (из логов):
val vin = "LDC623T20H1234567".toByteArray() // 20 байт

// Неправильный:
val wrongVin = "INVALID_VIN_NUMBER_123".toByteArray()
```

**Что произойдёт:**
- ⚠️ **Сервер может отклонить** (не совпадает VIN автомобиля)
- ⚠️ **В логе:** `VIN mismatch, rejecting packet`
- ❌ **Спойлер не сработает**

---

### 3. Некорректная контрольная сумма

**Пример:** Неправильная checksum
```kotlin
// Пакет без checksum или с неправильным
val packet = ByteArray(512)
// ... заполняем данные ...
// Не вычисляем checksum для bytes 482-511
```

**Что произойдёт:**
- ⚠️ **Сервер обнаружит ошибку** (checksum не совпадёт)
- ⚠️ **Пакет будет отклонён**
- 📝 **В логе:** `Checksum error, discarding packet`
- ❌ **Команда не выполнится**

---

### 4. Некорректный timestamp (bytes 24-29)

**Пример:** Старый timestamp
```kotlin
// Правильный: текущее время в BCD формате
// Неправильный: время из прошлого
val oldTimestamp = byteArrayOf(0x20, 0x21, 0x03, 0x14, 0x12, 0x00) // 2021-03-14 12:00
```

**Что произойдёт:**
- ⚠️ **Сервер может отклонить** (устаревший пакет)
- ⚠️ **Защита от replay-атак**
- 📝 **В логе:** `Timestamp expired`

---

### 5. Некорректный encryption mark (byte 21)

**Пример:** Неправильный флаг шифрования
```kotlin
// Правильно: 0x01 = encrypted
// Неправильно: 0x00 = не encrypted (если требуется шифрование)
packet[21] = 0x00 // Вместо 0x01
```

**Что произойдёт:**
- ⚠️ **Сервер может отклонить** (ошибка шифрования)
- 📝 **В логе:** `Encryption mark mismatch`

---

### 6. Некорректная длина (bytes 22-23)

**Пример:** Неправильная длина пакета
```kotlin
// Правильно: 0x0200 = 512 байт (big-endian)
// Неправильно: 0x0100 = 256 байт
packet[22] = 0x01
packet[23] = 0x00
```

**Что произойдёт:**
- ⚠️ **Сервер обнаружит несоответствие**
- ⚠️ **Обрезка пакета или ошибка парсинга**
- 📝 **В логе:** `Invalid packet length`

---

### 7. Отправка команды другому компоненту

**Пример:** Команда окну вместо спойлера
```kotlin
// Команда окну (Component ID = 0x03):
val windowCmd = byteArrayOf(0xBF, 0x11, 0x0D, 0x03, 0x27, 0x24, 0x20, 0x01)

// Пытаемся открыть спойлер (Component ID = 0x06):
val spoilerCmd = byteArrayOf(0xBF, 0x11, 0x0D, 0x06, 0x27, 0x21, 0x20, 0x01)
```

**Что произойдёт:**
- ✅ **Команда выполнится** (но для другого компонента!)
- 🪟 **Откроется окно**, а не спойлер
- ⚠️ **Непредсказуемое поведение**

---

### 8. Отправка недопустимого значения позиции

**Пример:** Позиция вне диапазона
```kotlin
// Для спойлера: 1=открыть, 2=закрыть
// Отправляем недопустимое:
val invalidValue = 0x05 // Или 0x00, 0x03, 0xFF
```

**Что произойдёт:**
- ⚠️ **Сервер может проигнорировать** (невалидное значение)
- 📝 **В логе:** `Invalid value for property`
- ❌ **Спойлер не сработает**

## 📊 Реакция сервера на ошибки

### Уровни обработки ошибок:

**Уровень 1: Парсинг пакета**
```
┌─────────────────────────────────────┐
│  Проверка структуры                 │
│  - Длина (512 байт)                 │
│  - VIN (20 байт)                    │
│  - Encryption mark                  │
│  - Timestamp                        │
│  - Checksum                         │
└─────────────────────────────────────┘
           ↓
    Если ошибка → отклонить пакет
```

**Уровень 2: Валидация команды**
```
┌─────────────────────────────────────┐
│  Проверка Function Bytes            │
│  - Component ID (существует?)       │
│  - Action Code (валиден?)           │
│  - Значение (в диапазоне?)          │
└─────────────────────────────────────┘
           ↓
    Если ошибка → записать в лог
```

**Уровень 3: Выполнение команды**
```
┌─────────────────────────────────────┐
│  Отправка в CAN-шину                │
│  - Проверка состояния компонента    │
│  - Проверка безопасности            │
│  - Физическое выполнение            │
└─────────────────────────────────────┘
           ↓
    Если ошибка → вернуть error response
```

## 🛡️ Встроенные механизмы защиты

### 1. **Защита от replay-атак**
- Проверка timestamp
- Уникальность каждого пакета
- **Результат:** Старые пакеты отклоняются

### 2. **Проверка VIN**
- Соответствие VIN автомобиля
- **Результат:** Чужие команды отклоняются

### 3. **Checksum**
- Контрольная сумма всего пакета
- **Результат:** Повреждённые пакеты отклоняются

### 4. **Проверка состояния компонента**
- Доступность компонента (не сломан?)
- **Результат:** Команда невыполнимой функции отклоняется

### 5. **Проверка безопасности**
- Скорость автомобиля (<25 km/h?)
- Передача (в парковке?)
- **Результат:** Опасные команды блокируются

## 💡 Рекомендации для приложения

### 1. **Валидация перед отправкой**
```kotlin
fun validatePacket(packet: ByteArray): Boolean {
    // Проверка длины
    if (packet.size != 512) return false
    
    // Проверка VIN
    if (!validateVin(packet)) return false
    
    // Проверка checksum
    if (!validateChecksum(packet)) return false
    
    // Проверка Function Bytes
    if (!validateFunctionBytes(packet)) return false
    
    return true
}
```

### 2. **Обработка ошибок**
```kotlin
try {
    sendSpoilerCommand(cmd)
    log("Command sent successfully")
} catch (e: SocketException) {
    log("Connection error: ${e.message}")
    showError("Нет соединения с автомобилем")
} catch (e: Exception) {
    log("Unknown error: ${e.message}")
    showError("Ошибка выполнения команды")
}
```

### 3. **Проверка состояния перед командой**
```kotlin
fun canOperateSpoiler(): Boolean {
    // Проверка подключения
    if (!isConnected) return false
    
    // Проверка скорости (если доступно)
    val speed = getVehicleSpeed()
    if (speed > 25) {
        log("Safety block: speed = $speed km/h")
        return false
    }
    
    // Проверка передачи (если доступно)
    val gear = getGearPosition()
    if (gear != GEAR_P && gear != GEAR_N) {
        log("Safety block: gear = $gear")
        return false
    }
    
    return true
}
```

### 4. **Логирование для отладки**
```kotlin
fun logCommand(cmd: ByteArray) {
    val hex = cmd.joinToString(" ") { "%02X".format(it) }
    log("Sending: $hex")
    
    // Сохранить в файл для анализа
    saveToFile("spoiler_log_${timestamp}.txt", hex)
}
```

## 🎯 Выводы

### ✅ Хорошие новости:
1. **Сервер устойчив к ошибкам** — не крашится от некорректных данных
2. **Валидация на уровне сервера** — отклоняет плохие пакеты
3. **Логирование ошибок** — можно отследить проблему
4. **Защита от случайных команд** — несколько уровней проверки

### ⚠️ Плохие новости:
1. **Нет явного error response** — просто тишина в большинстве случаев
2. **Сложно отладить** — не всегда понятно, почему не сработало
3. **Безопасность блокирует** — даже правильная команда может не выполниться

### 💡 Рекомендация:
> **Тщательно валидировать пакеты перед отправкой!**  
> Использовать логирование для отладки.  
> Проверять состояние автомобиля перед командой.

---
