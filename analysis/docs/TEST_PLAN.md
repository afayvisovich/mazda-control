## 🚀 Пошаговый план тестирования

### Шаг 1: Подготовка (5 минут)

**На автомобиле:**
1. Включить зажигание (или запустить двигатель)
2. Подождать загрузку системы (30-60 сек)
3. Убедиться, что головное устройство работает

**На телефоне:**
1. Включить **Отладку по USB**
2. Установить MazdaControl APK
3. Запустить приложение

---

### Шаг 2: Сбор базовых данных (5 минут)

**На ноутбуке выполнить команды:**

```bash
# 1. Проверить подключение
adb devices

# 2. Очистить логи
adb logcat -c

# 3. Начать сбор логов
adb logcat -v time > full_logcat.txt

# 4. Получить список процессов
adb shell ps -A > processes.txt

# 5. Получить список пакетов
adb shell pm list packages -f > packages.txt

# 6. Проверить сеть (без root)
adb shell cat /proc/net/tcp > proc_net_tcp.txt
adb shell cat /proc/net/unix > proc_net_unix.txt

# 7. Системные свойства
adb shell getprop > system_properties.txt
```

---

### Шаг 3: Тест в MOCK режиме (2 минуты)

**В приложении:**
1. Убедиться, что включён **TEST (MOCK)** режим
2. Нажать **OPEN** → спойлер должен "открыться" (анимация)
3. Нажать **CLOSE** → спойлер должен "закрыться" (анимация)

**Ожидаемый результат:**
- ✅ Кнопки работают
- ✅ Анимация движения (2.2 сек)
- ✅ Логи записываются

**Проверить логи:**
```bash
grep "MOCK" full_logcat.txt
```

---

### Шаг 4: Переключение в REAL режим (1 минута)

2. Наблюдать логи подключения

```
=== CONNECTING CONTROLLERS ===
🚗 REAL MODE: Подключение к автомобилю...
```

**Действия:** Перейти к Шагу 6

**Если подключение успешно:**

## 🔍 Диагностика проблем

### Проблема 1: ConnectException

**Лог:**
```
❌ ConnectException: Connection refused
Server refused connection on 127.0.0.1:32960
```

**Причины:**
1. Сервер не запущен
2. Неправильный порт
3. Блокировка

**Решение:**
```bash
# Проверить, слушает ли порт 32960
adb shell cat /proc/net/tcp | grep "80E0"  # 80E0 = 32960 hex

# Найти процесс на порту
adb shell su -c "netstat -tuln | grep 32960"  # если есть root

# Проверить процессы com.mega.*
adb shell ps -A | grep "com.mega"
```

---

### Проблема 2: SocketTimeoutException

**Лог:**
```
❌ SocketTimeoutException: timeout
Connection timeout after 5000ms
```

**Причины:**
1. Сервер есть, но не отвечает
2. Сервер завис
3. Сетевая проблема

**Решение:**
```bash
# Перезапустить головное устройство
# Или перезапустить сервис

# Найти PID сервиса
adb shell pidof com.mega.rmu

# Перезапустить
adb shell kill <PID>
```

---

### Проблема 3: Команды уходят, но спойлер не двигается

**Лог:**
```
✅ Command sent in 3ms
✅ Command OPEN sent (512 bytes)
```

**Но спойлер не реагирует.**

**Причины:**
1. Неправильный формат пакета
2. Сервер не распознал команду
3. Спойлер заблокирован (безопасность)

**Решение:**
```bash
# Сравнить с оригинальными пакетами
grep "Header (30 bytes)" full_logcat.txt

# Проверить ключевые байты
grep "Byte\[355\]" full_logcat.txt
grep "Byte\[363\]" full_logcat.txt
```

---

### Проблема 4: UnknownHostException

**Лог:**
```
❌ UnknownHostException: 127.0.0.1
```

**Причины:**
1. Проблема с сетью
2. localhost заблокирован

**Решение:**
```bash
# Проверить hosts
adb shell cat /system/etc/hosts

# Попробовать явно
adb shell ping 127.0.0.1
```

## 🧰 Дополнительные инструменты

### Скрипт для быстрого тестирования
```bash
#!/bin/bash
# test_mazda_control.sh

echo "🚀 Starting Mazda Control Test..."

# Очистить логи
adb logcat -c
echo "✅ Logs cleared"

# Начать сбор логов
timeout 300 adb logcat -v time > full_logcat.txt &
LOG_PID=$!
echo "📝 Logging started (PID: $LOG_PID)"

# Подождать подключения
sleep 5

# Проверить процессы
adb shell ps -A | grep "com.mega" > mega_processes.txt
echo "📊 Mega processes captured"

# Проверить сеть
adb shell cat /proc/net/tcp > tcp_connections.txt
echo "📊 TCP connections captured"

# Подождать тест
echo "⏳ Waiting for test (5 minutes)..."
wait $LOG_PID

echo "✅ Test complete!"
echo "📁 Files: full_logcat.txt, mega_processes.txt, tcp_connections.txt"
```
