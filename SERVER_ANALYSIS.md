# 🔬 Анализ серверов автомобиля

## 🎯 Обнаружено ДВА РАЗНЫХ сервера

### Сервер 1: AG35TspClient (ТЕЛЕМАТИКА)
**Адрес:** `172.16.2.30:50001`

**Назначение:**
- Внешний TSP (Telematics Service Provider) клиент
- Протокол GB32960 (китайский стандарт телематики)
- Связь с внешним сервером для удалённого управления
- Передача данных о состоянии автомобиля

**Файлы в HuronCarService:**
```
smali/com/mega/car/tbox/AG35TspClient.smali
  - HOST: "172.16.2.30"
  - PORT: 0xc351 (50001)
  - Использует GB32960Processor

smali/com/mega/car/tbox/processor/GB32960Processor.smali
  - Обработчик протокола GB32960
```

**Свойства TBox:**
```java
// TBox.java
ID_TSP_LOGIN_STATUS = 0x30000009      // Статус входа TSP
ID_TSP_REMOTE_CONTROL = 0x30000002    // Удалённое управление
```

**Статус:** ❌ **НЕДОСТУПЕН** в вашем автомобиле
- Постоянные `IOException`
- `no connect socket!`
- Это внешний сервер, который не настроен или недоступен

### Сервер 2: Локальный IPC (УПРАВЛЕНИЕ ФУНКЦИЯМИ)
**Адрес:** `localhost/127.0.0.1:32960`

**Назначение:**
- Внутренний IPC (Inter-Process Communication)
- Связь между UI и сервисом управления функциями
- Команды: окна, спойлер, климат, двери и т.д.

**Файлы в HuronCarService:**
```

```

- Успешные команды окон/спойлера
- Локальная связь без сети

## 🔍 Почему возникла путаница

### Изначальное предположение (НЕВЕРНОЕ):
1. Видим в логах `CallServerInterceptor` → думаем, это AG35TspClient
2. Видим порт 32960 → думаем, это "тестовая заглушка"
3. Находим в коде AG35TspClient с портом 50001 → думаем, это "правильный сервер"

### Реальность (ВЕРНОЕ):
1. **CallServerInterceptor** использует **локальный сервер 32960** для управления функциями
2. **AG35TspClient** использует **внешний сервер 50001** для телематики
3. Это **ДВА РАЗНЫХ** сервиса с разным назначением!
